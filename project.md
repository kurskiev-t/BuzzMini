### Пункты (реализуем по порядку или помечаем зависимости)

1. [x] **Разбор ~6 ГБ `.venv` и стратегии уменьшения**  
   - **Замер** (скрипт `tools/measure_venv.py`, запуск из корня: `python tools/measure_venv.py`): на референсной Windows-`.venv` почти весь объём — `Lib/site-packages`, типично **~6.0 ГБ** суммарно. Крупнейшие каталоги: **`torch` ~3.95 ГБ**, **`nvidia` ~1.63 ГБ** (в основном дубликаты **cuDNN** и **cuBLAS** рядом с тем, что уже лежит в **`torch\lib`** в wheel PyTorch cu126), далее **PyQt6**, **`av`**, **ctranslate2**, **onnxruntime** (~37 МБ — зависимость `faster-whisper`; нужна для Silero VAD при `vad_filter=True`, у нас по умолчанию VAD выключен, пакет всё равно тянется из метаданных upstream).  
   - **Сравнение с Handy:** маленький установщик за счёт **другого стека** (Tauri + Rust, `transcribe-rs` / ONNX и т.д., без PyTorch в дистрибутиве) — см. `Handy\src-tauri\Cargo.toml`. Наша ценность — **стабильный путь torch + CUDA**; размер `.venv` сопоставим с экосистемой PyTorch, а не с Handy.  
   - **Модели** по-прежнему в `models/` (или `BUZZMINI_MODEL_ROOT`); размер весов отдельно от `.venv` — это нормально.  
   - **Сделано в репозитории:** убран неиспользуемый в коде **`torchaudio`**; убрана прямая зависимость **`nvidia-cudnn-cu12`** (на Windows wheel torch уже содержит cuDNN/cuBLAS в `torch\lib`; на Linux cuDNN для GPU-тянет **сам** `torch`+cu129 через lock). В **`buzz_mini/cuda_setup.py`** на Windows и Linux в путь загрузки нативных библиотек добавлен **`site-packages/torch/lib`** до каталогов `nvidia/*`, чтобы не требовать дублирующих wheels. Ожидаемая экономия на Windows после **`uv sync`**: порядка **~1.6 ГБ** (папка `site-packages/nvidia` исчезнет, если её больше не тянет ни один пакет).  
   - **Проверка у себя:** удалить/пересоздать `.venv` или выполнить **`uv sync`** по обновлённому `uv.lock`, затем снова **`python tools/measure_venv.py`** и прогон приложения с GPU (короткая диктовка). Если CUDA перестанет подниматься — откатить коммит и написать в issue.
