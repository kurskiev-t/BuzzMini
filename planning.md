## 1.1 Горячие клавиши (PTT)

- Замена дефолта: **левый Ctrl** вместо правого (привычнее части пользователей).
- Дальше: гибкая настройка «как в Handy» — пользователь сам выбирает комбинацию.
  - Перед внедрением: проанализировать реализацию в **Handy** (UI выбора шортката, хранение настроек; у них стек Tauri/React + Rust — см. их репозиторий и `src-tauri`).
  - Затем аккуратно спроектировать перенос в BuzzMini (Qt + `pynput`), без поломки текущего chord-подхода.

## 3. Надёжность и распространение

- Единый предсказуемый способ запуска для пользователей (venv / один интерпретатор / будущий установщик).
- Fallbacks: GPU → CPU с понятным уведомлением; диагностика `torch.cuda` при старте (уже частично в логах).
- Структура проекта и поток «PTT → запись → транскрипт → вставка» — документировать (UML / sequence), см. отдельную задачу позже.

## 4. Размер установки (~9 ГБ: ~6 ГБ `.venv`, ~3 ГБ модели)

- Модели — ожидаемый объём.
- Уменьшение `.venv`: выбор зависимостей (CPU-only torch для тех, кому GPU не нужен), опциональные extras, отдельный «легкий» профиль сборки, документирование что именно раздувает wheel (PyTorch CUDA + cuDNN и т.д.).
- Сравнение с установщиком Buzz — зафиксировать целевой размер после упаковки.
